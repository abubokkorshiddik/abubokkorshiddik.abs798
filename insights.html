Understanding GATHER and making health estimates transparent
Md. Abu Bokkor Shiddik
📅 6 April 2026 | ⏰ 12:00 AM

Why this topic matters

In health research, we often produce numbers that directly influence decisions, such as national rates, trends, and future projections. These estimates guide policies, funding, and intervention strategies. However, an important question remains: how transparent are these results?

What is GATHER

GATHER stands for Guidelines for Accurate and Transparent Health Estimates Reporting. Introduced in 2016, it provides a structured way to report health estimates clearly and reproducibly. It is not a method, but a reporting framework that ensures transparency.

Why GATHER matters

  • Health estimates often rely on multiple datasets
  • Statistical or machine learning models are used
  • Assumptions and adjustments are required
  • Uncertainty must be calculated

Without proper reporting, results may be difficult to interpret and trust. GATHER helps ensure clarity and reliability.

GATHER reporting domains

Category Focus area Purpose
Study definition Population, time, geography Defines scope
Data inputs Sources and characteristics Ensures traceability
Data processing Cleaning and adjustment Improves reproducibility
Modeling Methods and assumptions Explains estimation
Validation Model checking Ensures reliability
Outputs Results and uncertainty Improves interpretation
Transparency Data and code sharing Supports replication

Example of reporting estimates

Indicator Estimate Uncertainty interval
Disease prevalence 12.5% (10.2% – 14.8%)
Mortality rate 145 per 100,000 (120 – 170)
Projection (2030) 18.3% (15.0% – 21.6%)

Practical tips

  • Report uncertainty with estimates
  • Explain model choice clearly
  • State assumptions explicitly
  • Document missing data handling
  • Include validation results

Why researchers should care

  • Governments allocate resources using these estimates
  • Agencies plan interventions
  • Global comparisons depend on reliable data

GATHER promotes transparent and trustworthy science. It helps ensure that research findings are clear, reproducible, and useful for real-world decisions.

Normalization vs. Standardization in data preprocessing

When preparing data for machine learning models, features often have very different ranges and units. For example, one feature might range from 1 to 1000 while another ranges from 0.1 to 1. Without scaling, models that rely on distances or gradient-based optimization can perform poorly because features with larger numeric ranges dominate learning. Normalization and standardization are two common feature scaling techniques used to address this problem.

What is normalization

Normalization scales data to a fixed range, usually 0 to 1 or sometimes −1 to 1. The most common form is min-max normalization, where each value is rescaled relative to the minimum and maximum values of the feature.

Min-max normalization equation:

Xnorm = (X − Xmin) / (Xmax − Xmin)

Here, X is the original value, Xmin is the minimum value, and Xmax is the maximum value of the feature.

After this transformation, values are in the range [0,1]. This is useful when features have unknown distributions or are not Gaussian. It is often used in distance-based algorithms like k-NN and K-Means clustering because it prevents features with larger ranges from dominating distance calculations.

What is standardization

Standardization (also called z-score scaling) transforms features so they have a mean of 0 and a standard deviation of 1. This centers the data and scales it relative to how spread out values are.

Standardization equation:

Xstd = (X − μ) / σ

Where X is the original value, μ is the mean of the feature, and σ is the standard deviation.

After standardization, the feature has a distribution centered at 0 with unit variance. This is especially helpful for gradient-based models such as Support Vector Machines, Logistic Regression, and Neural Networks, and also for techniques like PCA that are sensitive to feature variance.

Key differences

Feature Normalization Standardization
Scaling goal Fixed range (0 to 1) Mean = 0, Std = 1
Typical range [0,1] or [-1,1] Unbounded
Formula (X − Xmin) / (Xmax − Xmin) (X − μ) / σ
Best for Distance-based models Statistical & ML models
Outlier sensitivity High Lower

When to use which

Use normalization when:

  • Data does not follow normal distribution
  • Using distance-based algorithms
  • Need bounded values

Use standardization when:

  • Data is approximately Gaussian
  • Using regression, SVM, PCA
  • Model depends on variance

Practical tips

  • Normalization is sensitive to outliers, so be careful
  • Standardization is more stable when outliers exist
  • Always fit scaler on training data only
  • Apply same transformation to test data

Example in python

from sklearn.preprocessing import MinMaxScaler, StandardScaler

# normalization
norm_scaler = MinMaxScaler()
X_norm = norm_scaler.fit_transform(X_train)

# standardization
std_scaler = StandardScaler()
X_std = std_scaler.fit_transform(X_train)

Common misunderstandings

In statistics, normalization sometimes refers to standardization. But in machine learning, normalization usually means scaling to a fixed range, while standardization means z-score scaling.

Visual intuition

  • Normalized data stays between 0 and 1
  • Standardized data centers around 0
  • Spread depends on standard deviation
